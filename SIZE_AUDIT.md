## Ranked Root Causes Of Size Bloat

1. `.next.zip` alone consumes 1.25 GiB. This appears to be an archived copy of generated Next.js output and is not required by the source project.
2. `.next` consumes 1.08 GiB. Most of that is `.next\dev\cache\turbopack`, created by `next dev` and safe to regenerate.
3. `node_modules` consumes 516.56 MiB. This is normal for a Next.js project with native Next/SWC, Sharp, Tailwind, TypeScript, ESLint, Supabase, and `react-icons`, but it should not be committed.
4. `node_modules\.vite` consumes 39.15 MiB. This is a leftover Vite cache from the old React/Vite setup and is safe to delete.
5. `dist` consumes 5.38 MiB. It is an old Vite production build and duplicates source assets from `src\assets`.
6. `react-icons` is a significant dependency at 82.20 MiB installed size, and generated chunks show large icon-pack modules. It is used by many components, so removal requires refactoring imports/usages.
7. `src\assets\Busniess\grocery.png` is the largest required source asset at 1.69 MiB. This is not the main bloat but is worth optimizing later.
